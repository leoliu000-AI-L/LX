# 技能进化提示词能力

> 学习如何将普通提示词进化为高密度、高质量、高审美的专业级技能提示词

---

## 🎯 核心理念

### 什么是"技能进化"？

**技能进化** = 将简单的任务描述 → 转化为结构化、高密度、专业级的技能提示词系统

**进化维度**:
1. **信息密度**: 从简单描述 → 多模块高密度内容
2. **视觉风格**: 从无风格 → 明确的审美体系
3. **工作流程**: 从单步骤 → 完整的多步骤流程
4. **质量标准**: 从模糊 → 精确的质量检查清单
5. **模板化**: 从一次性 → 可复用的模板系统

---

## 📊 进化对比分析

### 维度1: 信息密度进化

**原始版** (❌ 低密度):
```
生成一张关于"窗户型材选择"的信息图
```

**进化版** (✅ 高密度):
```
生成一张关于"窗户型材选择"的高密度信息图
包含7个核心模块:
1. 品牌选择（7个品牌卡片）
2. 壁厚标准（4级刻度尺）
3. 系列选择（3层对比）
4. 隔热条材质（对比图）
5. 铝材型号（产品进化）
6. 回收铝识别（技巧清单）
7. 避坑清单（警告区）

每个模块包含:
- 具体品牌名/型号
- 精确数值参数
- 对比数据
- 可视化建议
```

**进化秘诀**:
- 固定模块数量（6-7个）
- 每个模块必须有具体数据
- 明确可视化元素
- 提供对比/等级/场景

---

### 维度2: 视觉风格进化

**原始版** (❌ 无风格):
```
用好看的风格生成图片
```

**进化版** (✅ 明确风格):
```
【实验室精密手册感 + 波普实验风格】

**配色方案** (精确色值):
- 背景: #F2F2F2（专业灰白/蓝图网格）
- 系统基色: #B8D8BE（柔和蓝绿）
- 高亮警告: #E91E63（荧光粉红）
- 标记高亮: #FFF200（柠檬黄）
- 线条艺术: #2D2926（超细炭褐）

**视觉元素**:
- 技术图表: 爆炸图、截面图、锚点
- 坐标系统: 精确标记（0.5mm, 1.8mm, 45°）
- 数据块: "Marker-over-Print"效果
- 符号: 十字准星、数学符号（Σ, Δ, ∞）

**禁止事项**:
- ❌ 可爱/卡通涂鸦
- ❌ 柔和色或通用纹理
- ❌ 空白空间
- ❌ 扁平矢量图标
```

**进化秘诀**:
- 指定精确色值（#开头的HEX）
- 明确视觉风格参考（实验室手册 + 波普实验）
- 列出必须包含的视觉元素
- 明确禁止事项

### 维度3: 工作流程进化

**原始版** (❌ 无流程):
```
生成图片
```

**进化版** (✅ 完整流程):
```
**6步工作流程**:

步骤1️⃣: 启动询问
- 主题: 你想要制作的干货主题是什么？
- 描述: 用1-2句话描述核心要点
- 数量: 希望生成多少张图片？（3-10张）

步骤2️⃣: 搜索素材
- 搜索小红书平台相关高赞笔记
- 调用知识库中的专业内容
- 提炼爆款内容的共性特征

步骤3️⃣: 提炼价值
- ✅ 实用性强：用户能直接用得上
- ✅ 稀缺性高：不是烂大街的内容
- ✅ 引发共鸣：戳中用户痛点

步骤4️⃣: 智能拆分
- 将价值点拆分为6-7个核心主题
- 每个主题包含具体数据/品牌/参数

步骤5️⃣: 生成内容
- 按模板生成高密度内容
- 包含可视化元素建议

步骤6️⃣: 用户确认 → 自动生图
- 用户确认内容
- 使用精确 Prompt 生图
```

**进化秘诀**:
- 分步骤，每步有明确目标
- 步骤之间有逻辑关系
- 包含用户确认环节
- 提供具体的输出格式

## 🔧 技能进化方法论

### 方法1: 模块化分解

**原理**: 将复杂任务分解为6-7个标准模块

**步骤**:
1. **识别主题**: 明确要解决的核心问题
2. **维度分析**: 从哪些角度分析问题？
3. **模块定义**: 每个模块包含什么内容？
4. **数据收集**: 每个模块需要什么数据？
5. **可视化设计**: 每个模块用什么形式展示？

**示例** - "窗户型材选择":
```
主题: 窗户型材选择

维度分析:
- 品牌维度 → 模块1: 品牌选择
- 质量维度 → 模块2: 壁厚标准
- 应用维度 → 模块3: 系列选择
- 材质维度 → 模块4: 隔热条材质
- 型号维度 → 模块5: 铝材型号
- 识别维度 → 模块6: 回收铝识别
- 避坑维度 → 模块7: 避坑清单
```

**模板**:
```
## [主题名称]

### 模块1: [维度1]选择
- 选项A: [描述]
- 选项B: [描述]
[... 6-8个选项]

### 模块2: [维度2]标准
- [等级1]: ❌ [描述]
- [等级2]: ✓ [描述]
- [等级3]: ✓ [描述]
- [等级4]: 👑 [描述]

[... 其他模块]
```

### 方法2: 配色系统化

**原理**: 使用精确的色值系统，而非"好看的颜色"

**步骤**:
1. **风格定位**: 确定视觉风格（实验室、手账、科技等）
2. **色彩角色**: 定义每种颜色的角色（背景、主色、强调色、警告色）
3. **色值选择**: 选择具体的HEX色值
4. **使用规则**: 定义每种颜色的使用场景

**示例** - "实验室精密风格":
```
风格定位: 实验室精密手册感 + 波普实验风格

色彩角色:
- 背景: 专业灰白/蓝图网格 → #F2F2F2
- 系统基色: 功能区块和数据区 → #B8D8BE（蓝绿）
- 高亮警告: 关键警告/最优选择 → #E91E63（粉红）
- 标记高亮: 关键词荧光笔 → #FFF200（柠檬黄）
- 线条艺术: 技术网格、坐标线 → #2D2926（炭褐）

使用规则:
- 粉红只用1处（最重要的数据点或警告）
- 黄色用于关键词高亮
- 蓝绿用于主要功能区块
- 褐色用于所有线条和文字
```

**模板**:
```
【[风格名称]配色方案】

**色彩角色**:
- 背景: [描述] → #[色值]（[名称]）
- 主色: [描述] → #[色值]（[名称]）
- 强调色: [描述] → #[色值]（[名称]）
- 警告色: [描述] → #[色值]（[名称]）
- 线条色: [描述] → #[色值]（[名称]）

**使用规则**:
- [主色]用于[场景]
- [强调色]用于[场景]
- [警告色]仅用于[场景]
```

### 方法3: 工作流程标准化

**原理**: 将模糊的任务转化为标准的N步流程

**步骤**:
1. **输入定义**: 明确需要用户提供什么信息
2. **流程分解**: 将任务分解为N个步骤
3. **输出定义**: 每个步骤产生什么输出
4. **质量控制**: 每个步骤如何检查质量

**示例** - "小红书内容生成":
```
输入定义:
1. 主题: 你想要制作的干货主题是什么？
2. 描述: 用1-2句话描述核心要点
3. 数量: 希望生成多少张图片？

流程分解:
步骤1: 启动询问 → 获取用户输入
步骤2: 搜索素材 → 搜索小红书高赞笔记
步骤3: 提炼价值 → 筛选实用/稀缺/共鸣内容
步骤4: 智能拆分 → 拆分为6-7个模块
步骤5: 生成内容 → 生成高密度内容
步骤6: 用户确认 → 确认后生图

输出定义:
步骤1输出: 用户完整信息
步骤2输出: 素材列表和数据
步骤3输出: 价值点列表
步骤4输出: 模块划分
步骤5输出: 完整内容
步骤6输出: 最终图片

质量控制:
步骤1: [ ] 用户信息完整
步骤2: [ ] 素材丰富度足够
步骤3: [ ] 价值点明确
步骤4: [ ] 模块6-7个
步骤5: [ ] 每个模块有数据
步骤6: [ ] 用户确认
```

**模板**:
```
**工作流程** (N步法):

步骤[N]: [步骤名称]
- 输入: [需要什么]
- 操作: [做什么]
- 输出: [产生什么]
- 检查: [如何检查质量]
```

### 方法4: 质量检查清单化

**原理**: 将"高质量"转化为可检查的清单

**步骤**:
1. **维度拆解**: 从哪些维度评估质量？
2. **标准定义**: 每个维度的标准是什么？
3. **检查方法**: 如何检查这个标准？
4. **优先级**: 哪些标准是必须的？

**示例** - "高密度信息图":
```
维度拆解:
1. 色彩
2. 线条
3. 模块
4. 张力
5. 密度

标准定义:
1. 色彩: 只用系统性色彩
2. 线条: 有精细网格线
3. 模块: 6-7个独立块
4. 张力: 大标题vs小参数
5. 密度: 每个模块有数据

检查方法:
1. 色彩: [ ] 只用粉、绿、黄、黑
2. 线条: [ ] 有网格线和坐标
3. 模块: [ ] 数量6-7个
4. 张力: [ ] 对比明显
5. 密度: [ ] 有品牌/数值/百分比

优先级:
必须: 色彩、模块、密度
重要: 线条、张力
```

**模板**:
```
**质量检查清单**:

[维度1]:
- [ ] [标准1]
- [ ] [标准2]

[维度2]:
- [ ] [标准1]
- [ ] [标准2]

[... 其他维度]

**优先级**:
- 必须: [维度1], [维度2]
- 重要: [维度3], [维度4]
```
