# Cross-Session Memory — Minimal Checklist

## On Session Start

1. Read `SOUL.md`
2. Read `USER.md`
3. Read `memory/<today>.md` (if exists)
4. Read `memory/<yesterday>.md` (if exists)
5. In main/direct session, read `MEMORY.md`

## During Session

Log only important items to `memory/<today>.md`:
- Decisions made
- User preferences confirmed
- Notable errors + fixes
- Follow-ups / TODOs

## End of Session (or periodic heartbeat)

1. Review last 1–3 daily files
2. Move durable facts into `MEMORY.md`
3. Remove stale/outdated long-term items

## Guardrails

- Do not store secrets/tokens in memory files
- Prefer concise bullets over verbose logs
- If unsure whether durable, keep in daily file first
